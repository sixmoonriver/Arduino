我想通过arduino promini制作一个车船模型用的控制板，配合遥控器使用 ，通过NRF24L01无线模块接收遥控器的数据传输，通过1306OLED显示屏（0.91寸，128\*32）显示遥控接收到的数据，本地的设置及运行数据，OLED的I2c地址为0x3c，控制电机的运行速度和方向，以及舵机的角度，同时要能兼容有刷和无刷电机的模式。通过pcf8574扩容输入和输出。

# 接线图（参考YaoDriverV3板）

1. 电机输出接口；
   1. 2路BTS7960电机接口4\*2；D3、D9左侧，D2、D10右侧控制；D5、D6为PWM；
   2. 同上复用的4路无刷电机接口；采用3、5、6、9输出，要确认舵机的库是否使用了内置的定时器，不同的引脚输出的频率是否一致（默认5、6脚输出1000HZ的PWM，3，9输出500HZ）；
2. 电池电压检测接口：A0，采用91K和33K电阻分压后通过10K电阻接到脚，需要通过电压的值判断电阻是2S，3S，4S；
3. 电池电流检测接口：A1；
4. 8路数字I/O接口输出（通过pcf8574扩展，**地址0x20**），数字输入0\~3复用BTS7960电流反馈，数字4\~7位为输出，对应遥控器的控制位；
5. PCF8574输入（地址为0x21）,初始化全部为输入引脚，上拉，P0接地低电平时进入设置模式；

# 主控板设置

## 设置项：

1. 遥控器使用频道，默认90；
2. 电池设定，1s,2s,3s,4s；
3. 使用有刷电机， 还是无刷电机；
4. 是否使用辅助驱动电机，即输出pcf8574的4\~7位（无法同时使用4位输出开关）；
5. 左右电机的PWM差值，调整硬件的偏差，避免行驶中偏向一侧，取值范围+-32，默认为0，对左侧电机的PWM值进行修正；

### 设置方式：

1. 闭合输入PCF8574 P0引脚对地的开关，进入设置状态，禁止所有电机的控制，显示屏顶端显示setting...；
2. 读取地址0x21的pcf8574的0\~3位的状态（默认上拉状态），P1位是设置项切换，P2位是对设置项进行+操作，P3位对设置项进行减操作，退出设置状态自动保存设置到arduino;

# 动力控制

当输入pcf8574(0x21) P7为高电平（默认），使用有刷电机，为低电平时，使用无刷电机；

当油门输出在回中位时停止，油门往前推前进，往后推倒退。

## 有刷电机

1. 前进：D2/D3同时输出高电平，D9/D10同时输出低电平，D5，D6为传过来的油门数据；
2. 后退：D2/D3同时输出低电平，D9/D10同时输出高电平，D5，D6为传过来的油门数据；
3. 转弯：行驶方向不变，根据遥控传过来的方向控制数据，改变左右PWM的差值，方向控制往左转时，右侧电机PWM保持不变，减少左侧电机的PWM数据，反之亦然；

## 无刷电机

1. 无刷电机使用双向电调，同样，在油门回中位时停止；
2. 前进：根据油门大小控制无刷电调；
3. 转弯，根据遥控传过来的方向数据，控制左右电调的控制脉冲宽度，方向往左转，右侧电机的脉冲宽度只受油门数据控制，减小左侧电调的脉冲宽度，降低左侧电机的转速；

# 直线行驶自修正

通过mpu6050模块，检测加速度数据，在直线行驶时修正两个电机的转速；

# 开关信号控制

从遥控器传过来的0\~4位开关信号，分别控制输出pcf8574（0x20）的P4\~P7

# 信息采集和回传

需要采集，

1. 电池电压A0；
2. 电流A1；
3. 实际输出的左、右电机的PWM数据；
4. 辅助信息，如mpu6050的三轴数据

# 代码要求：

1. 代码实现尽量使用Arduino官方的库，请先生成遥控器部的代码，代码要求中文注释，方便新手能看明白；
2. 代码开始的注释行必须注明使用到的库和版本，开发者等信息；
3. 需要有看门狗程序，防止程序跑飞；
4. 尽量避免使用延迟程序导致阻塞；

# 显示需求：

默认显示第一屏，可以根据遥控器传过来的控制位数据，当P6为低电平时显示第一屏，P6，P7同时为高电平时显示第二屏，P7为低电平时显示第三屏；

## 第一屏显示本地信息：

1. 首行居中显示local，
2. 第二行当前遥控器的频道，功率设置；
3. 第三行显示: Bat: 电池电压值；Motor：有刷显示BDC/无刷显示BLDC，
4. 第四行当前最终输出的PWM值；

## 第二屏显示远程信息：

1. 首行居中显示：from remoter
2. 第二行显示: YM: 油门值；FX：方向值；
3. 第三行显示：RY：右侧摇杆Y轴的值；RX：右侧摇杆X轴的值；
4. 第四行显示：INPUTSW：遥控器P0\~P7的值；
