# 2.4G

我想通过arduino promini制作一个车船模型用的遥控器，通过NRF24L01无线模块进行数据传输，通过1306OLED显示屏（0.91寸，128\*32）遥控器及被控制设备回传的相关的数据，OLED的I2c地址为0x3c，通过pcf8574扩展IO口，I2C地址为0x27

## 以下是输入引脚接线：

左x电位器中点接模拟IO A3；

左Y电位器中点接模拟IO A1；

右X电位器中点接模块IO A0；

右Y电位器中点接模块IO A2；

显示屏SDA   A4；

显示屏SCL   A5；

电池电压检查：A6，电池电压通过2个470K电阻分压后通过4.7K接入到A6

镜头控制右：A7

镜头控制左，没有使用，因为没有模拟引脚了。

数据IO: 原先的拨动开关变成了轻触式开关。

D2:  pcf8574中断输入

D6：LED显示，低电平LED亮，开机显示，电池低电压闪烁提醒；

D3、D4、D5未使用

### pcf8574做为做输入扩展：

1. P0--->左前轻触式开关；
2. P1--->右前轻触式开关；
3. P2--->中左轻触式开关；
4. P3--->中右轻触式开关；
5. P4右侧3档开关往内打低电平，打中间为开路，两侧开关均为上拉高电平，p4/p5同时只能有一个为低电平；
6. P5右侧3档开关往外打低电平，打中间为开路，两侧开关均为上拉高电平，p4/p5同时只能有一个为低电平；
7. P6左侧3档开关往外打低电平，打中间为开路，两侧开关均为上拉高电平，p6/p7同时只能有一个为低电平；
8. P7左侧3档开关往内打低电平，打中间为开路，两侧开关均为上拉高电平，p6/p7同时只能有一个为低电平；

### 以下NRF24L01的接线

D7 → CSN

D8 → CE

D11 → MOSI

D12 → MISO

D13 → SCK

### OLED接线

A4 → SDA

A5 → SCL

VCC  → 5V

GND → GND

### PCF8574接线

A4 → SDA

A5 → SCL

VCC  → 5V

GND → GND

## 遥控器设定

### 设置项

1. 遥控器使用频道，默认90；
2. 采样最大值simpleMax，针对所有摇杆，默认1024，即5V参照电压下的5V输入，要设最大值保护（if x >= simpleMax, x = simpleMax ），防止设置不合理导致的溢出；
3. 电池设定，1s，2s, 3s；
4. 油门最大值设定，限制最高速度，防止电池过载或者测试阶段；
5. 转向最大值设定，限制左右转弯的最大值，防止转弯过急或者用于差速转弯调试阶段，避免反复通过代码来调整；
6. 设置摇杆的正反，左/右，X/Y共4个；

### 设置方式：

1. 同时按压pcf8574引脚P0、P1所接的开关持续2秒中，进入设置状态，同时屏幕上端显示setting....， 显示当前的设置值；
2. 短按P0，会轮询显示需要调整的项，P2对所设置的值进行+，P3对所设置的值进行-操作；
3. 长按P0  2秒，保存设置到arduino，确保重启和掉电不会丢失，退出设置状态；

## 显示要求：

1. 每一屏显示4行，通过左侧的3档开关进行切换，P6为低电平时显示第一屏，P6，P7同时为高电平时显示第二屏，P7为低电平时显示第三屏；
2. 第一屏显示本地信息：
   1. 首行居中显示local，
   2. 第二行显示: Bat: 电池电压值，以0各1的形式显示几位开关的状态；
   3. 第三行显示几个摇杆的值和正/反；
   4. 当前遥控器的频道，功率设置；
3. 第二屏显示被控设备传回的数据；
   1. 首行居中显示：remote
   2. 第二行显示: Bat: 电池电压值，Cur: 电流值；
   3. 第三行显示左、右电机的最终输出值；
4. 第三屏为扩展输出，
   1. 首行居中显示: gps

## 代码要求：

1. 代码实现尽量使用Arduino官方的库，请先生成遥控器部的代码，代码要求中文注释，方便新手能看明白；
2. 代码开始的注释行必须注明使用到的库和版本，开发者等信息；
3. 需要有看门狗程序，防止程序跑飞；
4. 除setup函数之外，不能使用阻塞式编码，即禁止使用delay()函数；
